# Miri test selection has moved to .config/nextest.toml
#
# The [profile.default-miri] filterset defines which tests run under Miri.
# Nextest activates this profile automatically when MIRI_SYSROOT is set.
#
# To list which tests will run:
#   cargo miri nextest list -p minkowski --lib
#
# Selection criteria: tests that exercise raw pointer arithmetic, UnsafeCell
# access, atomic ops on mmap memory, type-erased column storage, arena
# allocation, or entity lifecycle through unsafe paths.
#
# Excluded from Miri:
#   - par_for_each (rayon thread pool unsupported by Miri, covered by TSan)
#   - Mostly-safe modules: access, view, transaction, debounce, tick,
#     lock_table, retention, planner (except execute_stream batch paths)
